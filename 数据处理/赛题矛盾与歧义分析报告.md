## 二、致命矛盾详析

### 2.1 成本参数完全缺失 — 问题2不可解

**题目声称**：附件表3包含"机器和人工集包成本和产能"

**数据实际**：附件表3列名为：

| 原始列名（GBK） | 解读后的实际列名 | 数值范围 | 实际含义 |
|---------------|----------------|---------|---------|
| 分拣中心 | 分拣中心 | — | 站点标识 |
| 分拣设备类型 | 分拣设备类型 | 分拣1~8 | 现有设备型号 |
| 人工建包成本 | **格口数量** | 24 ~ 698 | 该场地的格口数 |
| 机器建包成本 | **产能上限** | 0 ~ 200,000 | 日处理包裹上限 |

**证据**：
1. "机器建包成本"列：同一设备类型下所有场地的值**完全相同**（如分拣机1全部=100,000，分拣机2全部=200,000），这是设备产能特征，不是场地成本。
2. "人工建包成本"列：值范围24~698，与附件表4中设备格口数（100~600）量级一致。
3. 若将100,000理解为"机器建包成本（元/天）"，全网成本将达万亿级别，完全不合理。

**建模障碍**：
- 问题2目标：`min Σ(人工建包总成本 + 机器集包总成本)`
- 但数据中没有给出"每流向上人工建包的成本单价"和"每包裹机器集包的成本单价"
- **没有成本数据，目标函数无法定义**

**可行的解决路径**：
- 方案A：将问题2转为**最小化人工使用量**问题（机器产能给定，人工作为补充，目标是最小化人工处理的流向数）
- 方案B：从问题3反推成本——人工=90元/人/天÷5格口/人=**18元/格口/天**；机器成本从附件表4设备折旧推导
- 方案C：假设人工成本单价=附件表3中"人工建包成本"列的值（但其实此列是格口数）

### 2.2 ai参数（格口处理能力）缺失

**题目原文**：
> "若场地 i 每个格口最多可处理 ai 个包裹，则处理一个包裹量为 x 的流向需要占用 ⌈x/ai⌉ 个格口"

**问题**：ai 这个参数在附件表3和附件表4中均无直接定义。

**数据中的线索**：

附件表4（可采购的新设备）：
| 设备 | 格口数 | 设计产能（件/天） | 推导 ai = 产能/格口 |
|------|--------|------------------|-------------------|
| A | 200 | 80,000 | 400 |
| B | 300 | 100,000 | 333 |
| C | 600 | 200,000 | 333 |
| D | 100 | 30,000 | 300 |

现有设备（附件表3）：无直接的设备产能/格口对应表。

如果假设 ai = 附件表3"产能上限"列 ÷ "格口数量"列：
- 范围：0 ~ 789
- 均值：391
- 中位数：约 370

**可行的解决路径**：
- 方案A：取附件表4的平均值作为全局常数（ai ≈ 350件/格口/天）
- 方案B：对每个场地，ai = 产能上限 / 格口数量（从附件表3推导）
- 方案C：基于附件表4中与新设备格口数相近的型号反推

---

### 2.3 路由不唯一 — 违背题目基本假设

**题目声称**："这条走货路由，对于每一个首分拣到末分拣到流向，是唯一确定的"

**数据事实**：

| 指标 | 数值 |
|------|------|
| 原始行数 | 50,434 |
| 去重后唯一 (始,目,路由) 组合 | 10,248 |
| 数据重复率 | 98.97% 的行是重复记录 |
| 仅1条路由的OD对 | 5,811（74.0%） |
| **有2~6条不同路由的OD对** | **2,038（26.0%）** |

典型多路由案例（A60分拣 → A14分拣，6条不同路由）：
```
路由1: A60分拣-A14分拣
路由2: A60分拣-A14分拣-A18分拣-A14分拣  ← 自环异常！
路由3: A60分拣-A14分拣-A27分拣-A14分拣  ← 自环异常！
...
```

**这不仅是"有多条路由"的问题，有些路由还包含自环（经过目的站后再绕回）**。

### 2.4 高峰日产能严重超载 — 问题2可能无可行解

**发现**：在历史数据中，同一分拣中心在同一天的中转量 vs 该站的日产能上限对比：

| 分拣中心 | 日峰值中转量（件） | 日产能上限（件） | 超载倍数 |
|----------|------------------|-----------------|----------|
| **A56** | 179,126 | 30,000 | **6.0x** |
| **A46** | 617,997 | 200,000 | **3.1x** |
| **A45** | 402,686 | 200,000 | **2.0x** |
| A91 | 178,626 | 100,000 | 1.8x |
| A69 | 159,404 | 100,000 | 1.6x |
| A1 | 131,803 | 100,000 | 1.3x |
| A76 | 123,561 | 100,000 | 1.2x |
| A82 | 118,787 | 100,000 | 1.2x |
| A102 | 221,907 | 200,000 | 1.1x |

**含义**：即使所有包裹走最优路由（最小化中转），这些中心在高峰日的实际中转需求也远超其标称产能。A56分拣机4仅30,000产能但要处理最高近18万件，这不是"建包方案优化"能解决的——意味着**当前设备配置根本无法支撑现有货量**。

**可能的解释**：
- 产能上限 ≠ 硬约束，实际运营中可通过加班/外包等方式突破
- 数据中产能上限是"设计产能"，实际产能有弹性
- 某些包裹已在其他站完成集包，实际在中转站只需过站（不拆包）——这点题目没有明确

### 3.5 附件表3/4设备分类体系不统一

附件表3用 分拣机1~8（8种），附件表4用 A/B/C/D（4种），这是**两套不同的分类体系**。

产能数值的对应推测：
| 附件表3 现有设备 | 产能 | → | 附件表4 可购设备 | 产能 |
|-----------------|------|---|-----------------|------|
| 分拣机1 (27站) | 100,000 | ↔ | 设备B | 100,000 |
| 分拣机2 (26站) | 200,000 | ↔ | 设备C | 200,000 |
| 分拣机4 (30站) | 30,000 | ↔ | 设备D | 30,000 |
| 分拣机6 (5站) | 30,000 | ↔ | 设备D | 30,000 |
| **无** | — | ← | 设备A | 80,000 |

设备A（200格口/80,000产能）在附件表3无任何站点配置，可能是中间档位的可选升级。

分拣机4的问题：同一设备类型、30个站点出现了**5种不同产能值**（0/20000/25000/30000/45000），说明产能是场地配置参数，不是纯设备参数。

### 3.3 "混包"概念的双重解读

| | 解读A | 解读B |
|---|-------|-------|
| "混包"含义 | 将去往**不同下一站**的包裹放入同一袋 | 将去往**同一拆包站**的包裹混在一起（不管下一站是否相同） |
| 约束："同流向同袋" | 同一下一站必须同袋，不同下一站可以混 | 同一下一站必须同袋，且只有下一拆包站相同的才能混 |
| 约束："目的地相同" | 末分拣相同 | 下一个拆包站相同 |
| 来源 | 题目文字"多个下一分拣流向合并" | 题目文字"目的地相同...下一个拆包场地必须一致" |

建议按题目配图（图2）和约束逻辑推断：混包条件 = 同一建包站发出 + 下一拆包站相同 + 最终目的地相同。

### 3.4 A31分拣产能上限=0 但实为大型枢纽

附件表3中A31分拣：格口数量=97，**产能上限=0**

但这并非小型站——其实际运营规模巨大：
- 作为始发站：发出 **1,917,818** 件
- 作为目的站：接收 **4,075,894** 件
- 在路由中作为中转站出现 **1,404** 次

一个日处理量超2万件的大型枢纽，标称产能却为0——这不可能是纯人工站（97个格口的设备+0产能显然矛盾）。

可能解释：
- 数据录入错误（遗漏该站的产能值）
- 该站设备特别标记（实验设备或外包站）

处理建议：根据其97格口 + 分拣机4类型，参考同类型站推断产能为20,000~30,000。

### 4.2 货量趋势与增长假设矛盾

- 历史161天货量趋势 R² = 0.023，日增量仅 +0.045%/天
- 79.7% 的OD对 R² < 0.1（即绝大部分OD对无趋势信号）
- 题目却假设"每年增长20%"——与历史数据完全不符

这不是数据错误，而是题目有意设定的**场景假设**（投资决策需考虑增长）。

---

### 4.3 星期效应不显著

- 周一~周日全网日均货量差异 < 5%
- ANOVA检验 p = 0.93
- 可能数据已经过平滑或该业务无周末效应

**建模启示**：预测模型无需星期特征项。

## 六、论文建议：需明确声明的假设

在论文的"模型假设"部分，至少应列出：

1. **成本定义假设**：如何从附件表3/4/问题3中推导人工和机器的集包成本单价
2. **ai取值假设**：每格口处理能力的具体数值和推导依据
3. **混包定义假设**：精确定义混包的条件和约束
4. **多路由处理假设**：对26%有备选路由的OD对，选择哪条路由（最短？货量最大？）
5. **无路由OD处理假设**：对279个仅有货量无路由的OD对的处理方式
6. **A31处理假设**：产能为0的站如何建模
7. **南阳分拣中心假设**：无设备参数的中转站的参数赋值
8. **折旧假设**：设备成本按5年直线折旧计入1年总成本
9. **集包袋容量假设**：每个集包袋最大容纳包裹件数（如无限制，混包无意义）
10. **增长假设**：20%增长是每个OD均匀增长还是总货量增长

**建议对ai、人工成本单价、集包袋容量等关键假设做灵敏度分析**，这是竞赛论文的加分项。
