# A题：物流网络集包规则及设备优化 —— 要点总结与概念解释

---

## 一、问题背景

长三角地区某物流企业运营一个由 **92个分拣中心** 组成的转运网络，每天处理约 **660万件** 小件包裹（161天累计10.63亿件）。企业将小件包裹装入 **集包袋**（建包）运输以提升效率，并在下游站点剪开集包袋（拆包）取出重新分拣。

**优化目标**：在满足混包约束的前提下，确定最优的建包规则，使得：
- 问题1：准确预测未来货量（建模基础）
- 问题2：确定每个OD对在哪些分拣中心建包，以**最小化总成本**
- 问题3：考虑货量年增20%，决策设备购置方案

---

## 二、网络结构

### 走货路由（附件表1）
- 每条OD对（始发→目的）有一条固定的走货路由
- 路由是一串顺序经过的分拣中心，如：`A20分拣 → A6分拣 → A24分拣 → A88分拣`
- 共7849条OD对，路由长度2~6站不等
- 路由长度分布：2站(1222)、3站(3460)、4站(2795)、5站(353)、6站(19)

### 分拣中心评级
采用五维度加权评分（枢纽度35% + 流量度25% + 影响力25% + 始发10% + 末端5%）：

| 等级 | 数量 | 代表站点 | 角色 |
|------|------|----------|------|
| S级 | 1 | A20分拣 | 超级核心枢纽，连接77个站点 |
| A级 | 7 | A6/A24/A32等 | 一级枢纽，高流量 |
| B级 | 17 | A1/A7/A31等 | 二级枢纽 |
| C级 | 35 | 多数区域站 | 中等级别 |
| D级 | 32 | 以"D"开头的末端站 | 末端/始发小型站 |

### 货流类型
- **净发件型**（21个）：始发量 > 到达量
- **净收件型**（52个）：到达量 > 始发量
- **双向平衡型**（19个）：始发到达基本均衡

## 三、核心概念详解

### 3.1 建包（集包）

将多个小件包裹装入一个大集包袋中，袋口封好，贴上标签。建包后的集包袋作为一个整体在物流网络中转运。

**为什么要建包？**
- 减少分拣次数：袋子里的小包裹不用在每个中转站重新分拣
- 节省格口：一个袋子只占一个格口，里面可能有上百个包裹
- 降低操作成本：减少扫描、装卸次数
- 保护包裹：袋装运输减少破损

### 3.2 拆包

在指定站点剪开集包袋，取出包裹重新分拣。

一份建包决策 = 某个站建包 + 运输 + 某个下游站拆包。

### 3.3 建包路径（决策变量）

包裹从始发站出发，沿走货路由经过多个分拣中心。**建包路径**就是在路由上被选中执行建包的那些站的序列。

**举例**：路由 `A1 → A5 → A10 → A20`

可能的建包路径：
- **方案A**（直达）：A1建包 → A20拆包（中间不拆，干线直达）
- **方案B**（分段）：A1建包 → A10拆包（重新分拣）→ A10建包 → A20拆包
- **方案C**（逐站）：每个站都拆包重新分拣（无建包，纯零担）

建包路径是问题的**核心决策变量**，问题2要求输出每个OD对选择的建包流向分拣中心序列。

### 3.4 混包（Mixed Packaging）

**定义**：将发往不同"下一分拣"、但"下一拆包点相同"且"最终目的地相同"的包裹合并到**同一个集包袋**中。

**场景**：在某个分拣中心，来了一堆包裹，它们的下一站各自不同，但它们的建包路径一致（在同一个站拆包，去同一个终点）。

**举例**（杭州分拣中心）：

| 包裹 | 下一分拣 | 下一拆包点 | 最终目的地 | 能混？ |
|------|----------|------------|------------|--------|
| P1 | 绍兴 | 温州 | 温州 | ✅ |
| P2 | 宁波 | 温州 | 温州 | ✅ |
| P3 | 绍兴 | 温州 | 金华 | ❌ |

- P1和P2的下一分拣不同（绍兴 vs 宁波），但它们都在温州拆包，且最终都是温州 → **可以混包**
- P3也在温州拆，但终点是金华 → **不能和P1/P2混**
- 混包后袋子贴上"直达温州"标签走干线直达，袋内包裹不经过绍兴、宁波

### 3.5 混包的两条约束

**约束①**：同一下一分拣流向的包裹必须归入**同一个**集包袋（不能把一个流向的包裹拆散到多个袋子）

**约束②**：混包的包裹必须满足——下一拆包场地一致 + 最终目的地一致

### 3.6 格口与设备

- **格口**：分拣机上的物理槽位，每个格口对应一个流向
- **人工集包**：受限流向数（一个人同时管不了太多流向）
- **机器集包**：包裹量x需占用 ⌈x / a_i⌉ 个格口
- 格口是稀缺资源，建包决策必须受格口数量约束

**设备型号**：

| 设备 | 格口数 | 年产能(万件) | 购置成本(万元) |
|------|--------|-------------|----------------|
| A | 200 | 150 | — |
| B | 300 | 200 | — |
| C | 600 | 340 | — |
| D | 100 | 95 | — |

折旧期5年。现有设备为分拣机1~8（附件表3）。

## 四、三个子问题

### 问题1：货量预测
**任务**：根据附件表2（2025-12-01 ~ 2026-05-10，161天历史数据），预测每个始发-目的分拣对**未来7天**的小件包裹量。

**数据特点**：
- 7849条OD对 × 161天
- 日均660万件，中位数206件/OD-天，最大值135024件/OD-天
- 存在周期规律（周内波动）和趋势

**输出**：结果表1 `[日期, 始发分拣, 目的分拣, 包裹量]`

### 问题2：集包规则优化
**任务**：基于问题1的预测货量，确定每个OD对的最优建包流向分拣中心序列，使总操作成本最小。

**影响因素**：
- 建包/拆包的操作成本
- 格口约束（人工集包受限流向、机器集包受限格口）
- 混包约束
- 运输成本（干线直达 vs 逐站停靠）

**输出**：结果表2 `[始发分拣中心, 目的分拣中心, 建包流向分拣中心]`

### 问题3：设备投资决策
**任务**：假设货量每年增长20%，决策各场地应购置哪些设备及数量，并给出对应的集包规则。

**考虑因素**：
- 现有设备承载能力
- 新增设备A/B/C/D的选择与数量
- 设备折旧（5年）
- 购置预算（题目可能给定，需确认）

**输出**：结果表3（Sheet1: 设备购置方案 | Sheet2: 集包规则）

## 六、解决思路框架

```
问题1（预测货量）
  │
  ├─ 时间序列分解：趋势 + 周期 + 残差
  ├─ 候选模型：Prophet / LightGBM / LSTM
  └─ 逐OD对预测未来7天
        │
        ▼
问题2（集包优化）
  │
  ├─ 输入：预测货量 + 走货路由 + 设备格口
  ├─ 决策变量：每条OD的路由上哪些站建包
  ├─ 约束：格口限制、混包规则
  ├─ 目标：最小化总操作成本
  └─ 方法：混合整数规划 / 启发式算法
        │
        ▼
问题3（设备投资）
  │
  ├─ 输入：问题2方案 + 年增20%货量
  ├─ 决策变量：各场地购哪些设备、买几台
  ├─ 约束：预算、场地空间
  └─ 目标：5年内总成本（设备折旧+运营）最小
```
