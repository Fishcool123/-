# A题：物流网络集包规则及设备优化 —— 完整做题思路

---

## 一、题目到底在问什么

一家物流公司有 **92个分拣中心**，每天处理约660万件小包裹。包裹从始发站出发，沿着一条**固定的走货路由**（附件表1），经过若干个中转分拣中心，最后到达目的地。

在每个分拣中心，小包裹面临一个选择：**装袋（建包）还是不装袋**。装袋就是把去往相同方向的小包裹塞进一个集包袋，袋子作为一个整体运输，到下游某个站再**剪开（拆包）**。

这个选择影响成本：
- 如果只在起点装袋、终点拆袋 → 袋子太空，浪费分拣机的物理槽位（格口）
- 如果每个站都拆了重新装 → 浪费分拣机的工作能力

**题目就是要在三个层次上帮你做决策：**

| 问题 | 核心问题 | 输出 |
|------|----------|------|
| 问题1 | 未来7天每个OD对的包裹量是多少？ | 预测表 |
| 问题2 | 每个包裹在哪些站该装袋、哪些站该拆袋？ | 集包规则表 |
| 问题3 | 如果货量每年涨20%，该买什么设备、买几台？ | 设备购置方案 + 集包规则 |

## 二、核心概念（用人话解释）

### 2.1 OD对（首-末分拣流向）

"OD" = Origin-Destination。从A站出发，到B站结束，就是一个OD对。题目中，同一个OD对的所有包裹走**同一条路由**。

**举例**：`A20分拣 → A6分拣 → A24分拣 → A88分拣` 就是一个OD对（A20到A88）的路由。

### 2.2 建包路径（决策变量）

包裹沿路由走，**在哪些站建包**。这是题目最核心的决策变量。

**举例**：路由 `A→B→C→D`，可能的建包路径：
- 方案A：A建包 → D拆包（中间不拆，一路到底）
- 方案B：A建包 → C拆包 + C重新建包 → D拆包
- 方案C：每个站都拆包重装（等于不建包）

建包路径必须满足**唯一性**：同一个OD对的所有包裹，建包路径必须相同。

### 2.3 混包

在某个分拣中心，把去往**不同下一站**、但**在同一个站拆包**、且**最终目的地相同**的包裹，合并到同一个集包袋里。

**为什么要混包？** 提高袋子的装满率，减少格口占用。

**混包的两个约束**：
1. 去往同一个下一站的包裹，不能分散到不同的袋子里（同流向必须同袋）
2. 混在一起的包裹，必须满足：下一个拆包站相同 + 最终目的地相同

### 2.4 格口

分拣机上的物理槽位，就像快递柜的格子。一个格口对应一个"下一站方向"。如果一个流向包裹量很大，就需要占多个格口（⌈包裹量 ÷ 每格口容量⌉）。

格口是**稀缺资源**——一个分拣中心的分拣机格口数量有限。

### 2.5 人工集包 vs 机器集包

| | 人工集包 | 机器集包 |
|------|----------|----------|
| 限制 | 最大处理流向数有上限 | 格口数量限制 |
| 成本 | 每个流向有成本 | 每个包裹有成本 |
| 关系 | 机器产能不够时人工补充 | — |

**总成本 = 人工建包成本 + 机器建包成本** → 题目要求最小化这个和。

## 三、问题1：货量预测

### 3.1 数据概况

- 附件表2：161天历史数据（2025-12-01 ~ 2026-05-10）
- 约115万行记录，7849个OD对
- 日均总货量约660万件
- 单OD单日：中位数206件，最大135024件

### 3.2 数据特点与处理

**（1）数据质量检查**
- 检查缺失日期（每个OD对是否每天都有记录）
- 识别异常值（突然暴增/暴跌的货量）
- 注意：有279条OD对有历史货量但无路由记录，预测时需标记或剔除
- 注意：A31分拣的人工成本为0（异常值，问题3中需处理）

**（2）时间序列特征**
- **趋势性**：整体货量可能随时间增长或下降
- **周期性**：周内波动明显（工作日多、周末少），周期=7天
- **稀疏性**：很多OD对货量很小甚至为0，需要零膨胀处理

**（3）特征工程**
- 日期特征：星期几（dayofweek）、是否周末、月份、日期序号
- 滞后特征：前1天、前7天、前14天的货量
- 滚动统计：过去7天均值、过去14天标准差
- OD对特征：路由长度、始发站等级、目的站等级

### 3.3 可用模型（按推荐顺序）

#### 方案A：LightGBM（推荐首选）

**为什么推荐**：速度快、效果好、处理稀疏数据强、对异常值鲁棒。

```
对每个OD对或分组OD对训练模型：
输入特征：日期特征 + 滞后货量 + 滚动统计 + OD特征
输出：未来1天的货量
滚动预测7天（用预测值作为下一轮的特征输入）
```

**关键技巧**：
- 可对所有OD对用一个模型（用OD对编码作为特征），也可对相似OD对分组建模
- 使用对数变换处理货量的长尾分布：`target = log(1 + volume)`
- 交叉验证：时序交叉验证（按时间切分，不用随机切）

#### 方案B：Prophet

Facebook出品的时间序列预测工具，对周期性数据效果好。

```
Prophet(seasonality_mode='multiplicative', weekly_seasonality=True)
```

**优点**：自动处理周期、节假日、趋势变化，可解释性强。
**缺点**：7849个OD对逐个训练太慢。可以只对货量大的OD对单独建模，货量小的用聚合模型。

#### 方案C：简单统计基线（benchmark必须做）

- **naive预测**：直接用前7天同星期几的值
- **移动平均**：过去7天均值作为未来7天预测
- **指数平滑（Holt-Winters）**：考虑趋势和季节性的平滑

用这些简单方法作为基线，证明你的复杂模型确实更优。

#### 方案D：深度学习（LSTM/Seq2Seq）

适合货量大且模式复杂的OD对。但数据量有限（161天），容易过拟合，建议作为对比实验而非主力模型。

### 3.4 预测策略选择

| OD对类型 | 日均货量 | 建议方法 |
|----------|----------|----------|
| 高流量（top 20%） | >1000件/天 | 逐OD对单独建模（LightGBM/Prophet） |
| 中流量（中间30%） | 100-1000件/天 | 分组建模或全局LightGBM |
| 低流量（bottom 50%） | <100件/天 | 简单统计方法（中位数/移动平均） |

### 3.5 模型评估

- **指标**：MAE、RMSE、SMAPE（对称平均绝对百分比误差）
- **验证**：最后14天作为验证集，前面147天作为训练集
- **对比**：至少对比3种方法，证明你的选择合理

### 3.6 论文写作要点

- 展示预测效果最好的10条和最差的10条OD对，分析原因
- 可视化：某条典型OD对的实际值 vs 预测值曲线
- 汇总统计：所有OD对的平均预测误差

## 四、问题2：集包规则优化

### 4.1 问题建模

这是问题的核心。本质上是一个**带约束的组合优化问题**。

**给定**：
- 走货路由（附件表1）：每个OD对的固定路由
- 预测货量（问题1结果）：每个OD对每天多少包裹
- 设备参数（附件表3）：每个分拣中心的机器格口数、成本、产能
- 人工参数：每站人工建包流向数上限、人工成本

**决策**：
- 每个OD对的每条路由上，选择哪些站作为建包站
- 建包站之间形成"建包段"：建包站A → 拆包站B

**目标**：最小化总成本 = 机器建包成本 + 人工建包成本

**约束**：
- 唯一性：同OD对必须相同建包路径
- 同流向同袋：去往同一下一站的所有包裹必须进同一个集包袋
- 混包条件：混包的包裹必须在同一站拆包、同一最终目的地
- 格口约束：每站使用的格口总数不超过设备格口数
- 人工流向约束：每站人工处理的流向数不超过上限
- 产能约束：每站处理的总包裹量不超过产能

### 4.2 数学形式化

**符号定义**：
- $R_{od} = (s_1, s_2, ..., s_k)$：OD对 $od$ 的路由，$s_1$ 始发，$s_k$ 目的
- $v_{od}$：OD对 $od$ 的（预测）日货量
- $x_{od,i} \in \{0,1\}$：OD对 $od$ 是否在路由第 $i$ 站建包
- $\text{cost\_machine}_j$：第 $j$ 个分拣中心每格口的机器成本
- $\text{cost\_manual}_j$：第 $j$ 个分拣中心每流向的人工成本

**每个站约束**：该站所有建包的OD对，按下一站分组，每组占用格口数 = ⌈该组总货量 / 每格口容量⌉。

### 4.3 求解方法

#### 思路一：贪心启发式（先做）

```
对于每个OD对：
  方案A（全直达）：只在首站建包，末站拆包
  方案B（全分段）：每相邻两站都建包+拆包
  方案C（贪心）：从首站开始，尽量往后延伸建包段，直到碰到
            格口/产能约束，则在上一站拆包，然后重新建包
  选择三个方案中成本最低的
```

**优点**：简单、快速、可解释
**缺点**：不是全局最优

#### 思路二：混合整数规划（MIP）（进阶）

使用线性规划求解器（如 Gurobi、CPLEX，学术免费版可用）：

```
决策变量：
  - y_{od,i} ∈ {0,1}：OD对 od 是否在路由第i站建包
  - z_{od,i,j} ∈ {0,1}：OD对 od 的第i站建包的包裹是否在第j站拆包

目标函数：最小化 ∑机器成本 + ∑人工成本

约束条件：
  - 唯一性约束
  - 同流向同袋约束
  - 格口容量约束
  - 产能约束
```

**优点**：理论上能得到全局最优
**缺点**：变量多（7849条OD × 平均3-4个中间站），可能无法直接求解

#### 思路三：分层优化（最推荐论文写法）

```
第1层：确定建包段
  对每条路由，枚举所有可能的建包路径方案（实际上不多，路由最长6站）

第2层：分配格口和集包方式
  在给定建包方案下，计算各站格口占用和成本

第3层：全局调整
  对格口超限的站，调整经过该站的OD对的建包方案（拆段或合并）
```

这种分层方法更容易在论文中讲清楚，而且可计算。

### 4.4 关键计算细节

**格口占用计算**：
```
对于分拣中心C：
  将经过C的所有建包的OD对按"下一站"分组
  每组货量 = 该组所有OD对货量之和
  该组占用格口 = ceil(组货量 / 每格口处理包裹数)
  C总格口占用 = 所有组的格口之和
```

**成本计算**：
```
机器成本 = 格口占用数 × 每格口成本
人工成本 = 使用人工的流向数 × 每流向人工成本
```

### 4.5 论文写作要点

- 画一张物流网络示意图，标注关键枢纽
- 用一个小例子（如3站路由）完整演示你的算法
- 展示优化前后各站格口占用对比
- 分析哪些站是瓶颈（格口利用率最高的站）
- 讨论混包带来的成本节省

## 五、问题3：设备投资决策

### 5.1 问题理解

在问题2的基础上，考虑未来1年的变化：
- **货量每年增长20%**（所以1年后货量 = 当前 × (1 + 20% × 年化天数比例)）
- 可以购买新设备A/B/C/D来扩充产能
- 设备有购置成本，按5年折旧
- 当设备产能不够时，可以靠人工补（每人每天处理5个格口，日工资90元）

### 5.2 决策框架

**目标**：最小化 1年内总成本

**总成本 = 设备折旧（1年分摊）+ 设备运营成本 + 人工成本**

**决策变量**：
- 每个场地买哪种设备、买几台
- 对应新设备下的集包规则（建包路径重新优化）

### 5.3 设备参数

| 设备型号 | 格口数 | 年产能(万件) | 购置成本(万元) | 年折旧(÷5) |
|----------|--------|-------------|----------------|------------|
| A | 200 | 150 | — | — |
| B | 300 | 200 | — | — |
| C | 600 | 340 | — | — |
| D | 100 | 95 | — | — |

注：具体数值需从附件表4读取。

### 5.4 求解方法

#### 步骤1：计算各站需求

根据问题2的集包规则 + 货量年增20%，计算每个分拣中心所需的格口数和产能。

#### 步骤2：识别瓶颈站

哪些站的现有设备无法满足需求？缺口有多大？

#### 步骤3：设备选型模型

对每个瓶颈站，这是一个小型背包问题：
```
该站需要额外G个格口、P件产能
设备选择：选哪种、选几台，使(折旧+运营)成本最小
约束：格口够用、产能够用
```

#### 步骤4：全局联动优化

改变一个站的建包规则会影响上下游，所以需要：
- 更新设备后的集包规则重新优化（迭代步骤1-3）
- 考虑设备富余格口是否可以承接更多建包任务（减轻上游压力）

### 5.5 投资策略原则

- **大枢纽优先**（S级、A级站）：货量大，设备利用率高，买好设备划算
- **中等站**：买中等设备 + 人工补充峰谷
- **小站**：纯人工或买最便宜的D型设备

### 5.6 论文写作要点

- 展示设备购置前后的成本对比
- 展示各站新增设备分布图
- 敏感性分析：货量增长率从15%到25%变化时，设备方案如何调整
- 讨论"买设备 vs 用人工"的盈亏平衡点

## 八、踩坑提示

1. **附件编码**：CSV文件是GBK编码，不是UTF-8，读取时注意
2. **A31异常**：A31分拣的人工成本为0，需要特别处理（可能是数据错误或免费人工）
3. **路由缺失**：279条OD对有货量无路由，这些OD对在问题2中无法确定建包路径，需说明处理方式
4. **货量0值**：很多OD对的某些天货量为0，预测模型需要能输出0（别用log后无法还原的模型）
5. **计算量**：7849个OD对 × 161天，处理时注意内存，建议分批处理
6. **不要过度复杂化**：问题2的MIP可能无法求解，优先用启发式方法保证可解性
